20-21 term 1 课程学习回顾

PhD 一年级的第一学期刚刚结束了。按照学校的培养计划，我完成了三门主修课程，一门研讨课，还有一些老板布置的事情。按理说 PhD 的主要任务需要集中在科研上，但这学期的三门课让我感受到了十分的压力，很出乎我意料。于是我觉得有必要记录下来，供我自己回顾，也可能可以为后来选择这些课的同学提供一点线索。
我所在的项目是 SMU 的 PhD Programme at School of Information System 。第一学期可供选择的研究生课程是：

  • IS706: Software Mining and Analysis
  • IS711: Learning and Planning in Intelligent Systems
  • IS712: Machine Learning
  • IS713: Foundations for Data Analytics
  • IS715: System Security

由于我的研究方向和 Security 与 Software 关系不大， 所以排除掉 IS706 和 IS715 之后，只有三门课留给我选。于是，接下来我想小结一下这三门课留我的印象和我认为比较合适的学习策略。

IS711

一门从名称上猜不出来要学什么的课程，以至于有同学跟我说被这门课的名称「骗」了过来。其实这门课主要讲了 Reinforcement Learning (RL) 这个大主题。除了基础铺垫之外，核心内容集中在 Markov Decision Problem (MDP)，Q-learning，和 Game Theory。我之前有困惑 MDP，Q-learning，Game Theory 的关系，老师为什么把这些内容放到这门课程中。这篇文章（Reinforcement Learning and Game Theory - An Intuitive Understanding）解答了我的疑惑。这三个主题同属 RL 框架下，作为不同的方法，解决不同的问题。（回过头来看，章节内容和课程目标不难理解，但是 Prof 安排课程从 Intelligent System 切入，前期弱化了 RL 这个范式，让我对课程结构产生了困惑。）
课程组织（成绩组成）比较简单，包含课堂发言，作业（Assignment），小测（Quiz），论文分享（Presentation），和期末测试（Exam）。
其实我能够分配给这门课程的时间很有限，所以最开始我采取了「理解不了就先背下来」的策略，希望通过背记先熟悉基本概念，在 quiz 和 assignment 复习的过程中再加深对内容的理解。另外，也因为暑假看了 Ali Abdaal 讲的 Active Recall 这个方法（用来记知识点），急迫地想在这门课程上实验一下。
课程开始一段时间后（其实挺长一段时间了，大概半学期的样子），我发现这个策略和这门课的特点不匹配。因为这门课程的内容和概念并不多，Prof 为了让我们理解，节奏不快，授课中夹杂手推公式和算例分析。我在这样的情况下强调先记住基本概念而不把重心放在概念的使用和理解，其实效率是不高的。实际经验也证明，如果能充分理解概念，这门课需要记忆的内容很少。从 quiz 和 assignment 的反馈来看，Prof 更加注重我们对概念的充分理解，并且考察我们对概念的拓展能力。举个例子，有一题这样问：

Consider you are a Uber driver, and of course you have learned this course. How can you enhance you reward by the approaches you learn from this course?

以上不是题目的全貌，但是能够看出 Prof 没有指出要求使用的方法，而其实这道题言之有理即可。
半期之后，我调整了策略，在每次复习的时间段尽可能的理解这个概念。有多余的时间，我就去看一些资料。这样做可能会导致进度较慢，每理解一个知识点的成本都比之前那种策略要高。但因为总的知识点不多，这个方法是走的通的。而且深挖概念理解的一个好处是，我们还会做一些对比学习，这使得我们对某一块的知识掌握更全面。这些拓展出去的知识，可能就会成为考试的一个考点。举个例子，期末有这样一道题：

(description of a Nash Equilibrium Game) ... , Please provide a mixed strategy.

关于纳什均衡的混合策略（Mixed Strategy）部分，Prof 在课件里面只提过纳什均衡的策略可以是纯策略（Pure Strategy）和混合策略。结果混合策略出成了一个大题。
总结：对于知识点不多但有一定理解难度的课程，应在第一次学习和随后的小测复习中强调概念理解和应用。Active Recall 在这种课程中帮助有限，可以在期末复习中使用。

IS712

呼，这门课就要直接很多。首先，这是一门 Machine Learning 课程，他会像所有的机器学习课程一样，介绍监督学习，非监督学习。此外，这是由 Hady W. LAUW 授课的课程，他有一些 Prof. Lauw 的个人特色。在 Welcome Mail 里面，Prof. Lauw 这样写：

This is a class intended for postgraduate research students. As such, the emphasis will be on fundamental concepts and underpinnings of machine learning. There will also be an exposure to accessing the relevant machine learning literatures. Moreover, you are also expected to be independent learner, capable of taking responsibility for your own learning.

上面一段话给这门课程打上了注脚：1. Prof. 关注基本概念，所以这门一学期的课程会覆盖机器学习大部分话题，内容非常多（举例就是 CNN 在一节课结束）；2. 这门课面向研究生，要把你们暴露在文献下面（就是会读很多论文）。Prof. 这样说了，他也是这样做的。
课程组织相对 IS711 就显得更加丰富和全面。课程包含讲授（Lecture），5 次小测（Quiz），论文分享（Presentation），课程项目（Project），课程参与（班级在 Piazza 上面有个讨论组，每个学生的发言会被统计并且转化成 Participation 分数），期末测试（Exam）。因为每周 Presentation 的论文都被放在 Pizza 上面讨论，所以实际上每周还有两篇论文阅读并提问。BTW，Prof. 在第一节课推荐我们每周分配 12 小时在这门课程上（我没做到 ... 2333）。
我自认为这门课我并没有学好，原因之一是我每周确实拿不出 12 小时给它，另一个原因是学习策略上出了问题。如此来看，我的学习经历并没有太多的发言权。我就结合期末测试题目，分析一下我认为更好的学习策略吧。
这里先给出策略：

  1. 课件讲了多少，就学多少
  2. 记忆基本概念，辅助理解（aka, active recall）

第一点策略字面意思有些消极，需要稍加解释。「讲多少就学多少」是站在完成这门课的角度来说。由于课程包含大量的内容，而 machine learning 的每个 topic 实际都可以延伸许多内容出来。比如讲 CNN，Coursera 上对应课程需要 20 小时完成，单看授课时间，也不可能在两个小时内完成。所以 Prof. 有侧重的选择了讲解内容，在有限的时间里，我们先掌握课件里的内容是学习这门课的一个次优解了。不过如果有多余的时间能够分配给这门课程，我们当然可以进行更多的探索。
第二点是我在学习新的领域，特别是引入许多新概念的领域，常采用的策略。在完成第一遍学习之后，将课程里面的知识点转化为一个个问题，并且定期的让自己回答这些问题。这样一个过程其实相当于一次次地将大脑里面的知识提取出来，经实验证明，这样记忆知识是最有效的，具体解释可以去看 Ali 的视频。所以对我来说，第一遍学习我对知识有基本理解，随后我通过记忆增加对知识的熟练度，对知识的熟悉又会加深我对知识的理解。重复「理解-记忆」这个循环几次之后，相应的知识就被牢靠的掌握了。
总结：对于知识点多的课程，优先掌握课件内容，配合「理解-记忆」的循环过程，能够在有限的时间里，完成课程要求。完成了课程要求的基础上，我们在延伸学习时就更从容了，学习效果也更好一些。

IS713

这门课没有考试，我就讲一下我的体验好了。课程主要讲经济相关的计量分析，参考书是 Introduction to Econometrics by James H. Stock。老师会按照随书课件把计量相关的分析方法都讲一遍。这门课的特点是作业很多，至少留两天的时间吧。课程要求做一个项目，这个项目会把课程里面学到的所有知识串起来，我觉得很有帮助。同时让我复习了一遍 R（R 处理数据真的比 Python 顺手）。
课程组织包含讲授（Lecture），作业（Assignment），项目（Project）和论文分享（Presentation）。
