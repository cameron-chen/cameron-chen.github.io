AHK 第一辑：Win 上的效率工具

2019-04-07 · 11 min read
ahk 效率

快捷键 (shortcut) 是计算机系统一个重要组成部分。你想想熟悉的ctrl + C和ctrl + v，是不是已经融入你的操作习惯中了。有一次我在学校图书馆办借书证，老师帮我录入信息。她的复制粘贴操作是选中文字 --> 点击鼠标右键 --> 选中复制 --> 切换到另一程序 --> 点击鼠标右键 --> 选中粘贴，而你会怎么做？鼠标点击的动作由键盘上的组合按键完成，且我认为很重要的一点：手不用移开键盘去控制鼠标（或触摸板）。对，这很重要。

顺着ctrl + C这条小逢，向快捷键的大门窥一窥。想想，如果：

  • 两个按键就能让你启动某个程序，而不是在桌面上铺一堆图标
  • 两个按键就能让你打开一个网页，而不是打开浏览器，再去收藏夹
  • 两个按键就能让你完成一系列重复工作，比如新建文件夹，而不是机械的重复
  • 用键盘完成文件编辑，不需要鼠标，甚至忘了方向键
  • 用/qq就能快捷输入 QQ 号。同理，你可以设定所有需要重复输入的账号、邮箱、特定字符串
  • ...

上述功能由 AutoHotKey 实现。其本质上是一个编程软件，但创造它的初衷是能够方便地用按键组合实现操作，所以几乎没有门槛，软件带来的收益却让人十分满意。个人认为这是一项 “20% 付出带来 80% 收益” 的投资。

实现环境

  • 软件：AutoHotKey_L，直接下载最新版本的exe文件
  • 操作系统：Win10
  • 技术基础：windows 基本操作，能通过搜索引擎排除简单错误
  • 帮助文档：AutoHotKey Docs

安装运行

本着不再造车轮的原则，安装及设置见：Win下最爱效率神器: AutoHotKey。

但是这里仍有几个和程序运行相关的建议：

  • 多看帮助文档。由于软件小众的原因，攻略也是零零碎碎，最好的学习方式是直接看帮助文档，命令的用法和实例很详细。上文提供的帮助文档是官方文档的中文版，适用于最新版本的 AutoHotKey_L
  • 程序版本的差异。由于程序的维护者几经变化，以及开源的原因，AutoHotKey 也在不停发展，且根据需要产生了不同的分支。在选择软件或者搜寻攻略的过程中可能会引起误会。这里推荐一般用户使用AutoHotKey_L，下载地址：https://github.com/Lexikos/AutoHotkey_L/releases

热键

Wiki 定义：

快捷键，又称为快速键、组合键或热键，指通过某些特定的按键、按键顺序或按键组合来完成一个操作，很多快捷键往往与如 Ctrl 键、 Shift 键、 Alt 键、 Fn 键以及 Windows 平台下的 Win 键和 Mac 机上的 Cmd 键（曾有 Meta 键）等配合使用。

通过 AutoHotKey 可以实现运行程序，打开网页和更加复杂的预设操作。

启动程序

自定义组合按键启动常用程序非常方便，是 AutoHotKey 设计之初的基本功能之一，所以能用非常简单的语句直接实现。

启动程序的命令是 run，基本的语法为：

[组合键]::run [程序路径]

语法很好理解，下面列出我的快捷启动列表：

; Capslock 表示键盘大写锁定键 Capslock
; & 表示连接前后两个按键，无对应按键
; o, r, w 等表示键盘对应字母按键，可以根据软件名称设置对应的字母，方便记忆
; 如 OneNote 我就使用了 OneNote 首字母 o 作为组合键
; 注 1：只要满足语法要求的组合键都可以作为触发程序的 [组合键]，这里我自己习惯用 Capslock + [字母]
; 注 2：分号 “;” 引导注释，引导的内容 AHK 不会读取
; 注 3：AHK 只识别半角标点，即需要用英文输入法输入标点。如：能够识别 “.”，但不能识别 “。”

Capslock & o::run C:\Program Files\..\Office16\ONENOTE.EXE
Capslock & r::run D:\Program Files\..\Everything\Everything.exe
Capslock & w::run C:\Program Files\..\WeChat\WeChat.exe
Capslock & h::run C:\Program Files\..\HandShaker\HandShaker.exe
Capslock & b::run C:\Program Files\..\BaiduYunGuanjia.exe
Capslock & m::run D:\Program Files\..\XMind\XMind.exe
Capslock & y::run D:\Program Files\..\Yu Writer\Yu Writer.exe

在设置完这步之后，你的桌面就不需要程序图标了。

触发网页

AutoHotKey 能够在浏览器没启动的情况下自动打开浏览器并跳转到预设网址。如果浏览器已经开启，则在新标签页打开网址。

同启动程序，启动网页有几乎一致的语法：

[组合键]::run, [网址]

下面列出我的快捷启动列表：

; # 表示键盘上的 win 键，也就是 windows 的标志
; # 和字母组合不再需要 & ，这是语法规则
; 同理，网站也可以自定义字母，达到方便记忆的目的

#b::run, https://www.baidu.com/
#g::run, https://www.google.com/
#m::run, https://mubu.com/
#s::run, https://sspai.com/
#w::run, https://www.weiyun.com/
#a::run, https://airtable.com/

触发程序

这里的程序不是指运行exe程序，而是指一系列的步骤，或者叫脚本（Script）。能够完成更加复杂的操作。

语法会复杂一点：

[组合键]::
[一系列操作]
return

下面列出我的快捷动作实例：

; 对于高阶用户，AHK 是一个完整的编程软件，甚至能够用来操作 excel 制作图标这样较复杂的动作
; 这里我只举一例，更多用法会在下期展示

; win + f 快捷新建文件夹
#f::            ; 用 win + f 触发动作
Click right     ; 模拟点击鼠标右键
Send, wf        ; 模拟点击键盘 w 和 f 键
Sleep, 125      ; 避免操作过快，等待 125 ms
return

文本编辑

AutoHotKey 能够通过脚本实现方便的文本编辑操作，实现逻辑和触发程序是一样的。

文本编辑对于在使用电脑的过程中不可回避，不管是编写报告还是写程序。文本编辑的过程中，光标的移动，翻页等，在以往我喜欢通过鼠标实现，键盘基本上就是作为输入文字的工具，但这个过程中，手要在键盘和鼠标之间不停切换，这其实是个很低效的事情。你可能以往习惯这样的操作，觉得其实也还好。但是我敢说你要是尝试过用快捷键完成光标移动这些操作，手放在键盘上就能做所有事，就回不去了。

Vim是Linux上的编辑器，在不能用鼠标的 Terminal[1] 上，程序员们又体现了他们的创造力，仅仅通过键盘也能高效地进行光标移到，翻页跳行，复制粘贴[2]。根据 Vim 的启发，结合我自己使用习惯，我定义了这样一批适合我文本编辑快捷键。对于重度文字编辑工作者来说，这些快捷键带来的是幸福感的直线提升。

下面列出我的快捷动作实例：

; ! 表示键盘上的 alt 键
; ^ 表示键盘上的 ctrl 键

!1::Send, {end}        ;到行末
!2::Send, {home}       ;到行首
!j::send, {left}       ;光标左移一格。这里用 alt + j, l, i, k 实现方向上下左右移动，手就不需要到
!l::send, {right}      ;光标右移一格。右下角去寻找方向键
!i::send, {up}         ;光标上移一格
!k::send, {down}       ;光标下移一格

!d::   ;删除一整行文字
Send {Home}            ;输入 home 键
Send +{End}            ;输入 shitf 键 + end 键
Send {delete}	       ;输入 delete 键
return 
!x::   ;选中一整行文字
send {home}
send +{end}
return 
!y::   ;复制一整行文字
send {home}
send +{end}
send ^c                ;输出ctrl+c，复制一整行
return
!r::   ;用剪切板内容替换一整行文字
send {home}
send +{end}
send ^v                ;输出ctrl+v,粘贴
return
!p::   ;另起一行粘贴剪切板内容																				
send {end}																					
send {enter}																				
send %clipboard%	   ;将剪贴板的内容输出													
return

热字串

定义：

热字串概念对标热键，即通过自定义字符来替换预设字符。

说人话就是用相对短的一句话替换相对长的一句话，来达到节省时间，提高效率的目的。如：输入 “/qq” 则程序帮助你输入 QQ 号。这样的快捷输入对于会长期重复使用的文字来说是很方便的选择。

常规输入

热字串是 AutoHotKey 发展过程中，对标热键发展出来的另一功能。

其基本语法为：

::[代替字符串]::[被替代字符串]

下面列出我的热字串实例：

; “/” 并不是有特殊功能的按键，只是平时不太会用到 “/ + 字母” 这样的输入，那这样输入的时候一定是希望触发热字串。

::/xh::314010****               ; 学号
::/qq::40812****                ; qq 号
::/qm::40812****@qq.com         ; qq 邮箱
::/wm::1781686****@163.com      ; 163 邮箱
::/sj::1781686****              ; 手机号

以上代码，键入/xh 后，再加空格、或 tab、或回车，就可以触发缩写；根据输入不同方式（空格，tab，回车）输出的内容后也相应附加了 [空格] [tab] [回车]。在你熟练之后，一直用一直爽。

获取日期并输入

热字串除了替代固定内容，还能根据时间等不同情况，替代动态内容。比如：用热字串输入当前日期。

下面列出我的热字串动态实例：

; 这里提供实例：热字串输入当前日期。更多应用可以自己探索。

::/dd::
d = %A_YYYY%.%A_MM%.%A_DD%    ;获得系统时间比如今天的时间：2007.10.21。如果需要其他格式请替换上面的“.”，如获得时间：2007-10-21
clipboard = %d%    ;把 d 的值发送到剪贴板，变量是不用声明的，想引用变量的值，就在变量的前后加“%”。第二行的变量是 AHK 自带的变量
Send ^v
return

附录

这里附上 AutoHotKey 语法中的常见映射关系：

符号 说明
# Win 键
! Alt 键
^ Ctrl 键
+ Shift 键
& 组合任意两个按键，或者鼠标按键
:: 分隔键，起分隔作用，组成语法结构中的一部分
; 引导注释，其引导内容 AHK 不会读取
run 表示运行某一程序，或跳转到某一地址

更多内容见帮助手册。
